# 数据库答辩速查卡

---

## 一句话概括
> 我负责的是 ShopCompare 跨平台比价系统的**数据库设计与数据管道**，基于 SQLAlchemy ORM，支持 PostgreSQL/SQLite 双模，爬虫数据自动采集、去重、聚合，支撑前端 36 个 API 端点。

---

## 核心数字
- **6 张表** / **646+ 商品** / **660+ 平台价格** / **733+ 历史记录**
- **4 个数据平台** / **6 大品类** / **18 个爬虫关键词**
- **36 个 API 端点全部通过测试**
- **爬虫每 30 分钟自动更新**

---

## 三个关键词

### 1. 双模降级
`DATABASE_URL` 环境变量 → PG；不设 → SQLite 零配置跑起来

### 2. 数据管道
爬虫 → 去重匹配 → upsert写入 → 实时聚合价格 → 六维评分

### 3. 测试隔离
独立 test DB + dependency override，test 和生产互不污染

---

## 遇到并解决的问题

| 问题 | 解决 |
|------|------|
| PG 连接密码认证失败 | 改用 Docker 中已有的 kes-postgres 容器 |
| 团队协作需要零配置 | 环境变量降级 SQLite |
| Demo 和爬虫数据混淆 | 归档 demo，全用爬虫数据 |
| 合并冲突(远程又加了demo) | 坚持爬虫专属方案，正确合并 |
