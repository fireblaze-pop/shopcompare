## 一、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| ORM | SQLAlchemy 2.0 | Python 最主流的 ORM |

| 部署 | Docker Compose | 一键启动 PG 服务 |

## 四、核心技术亮点

### 4.1 PostgreSQL / SQLite 双模降级

```python
# database.py
DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///./shopcompare.db")
if DATABASE_URL.startswith("sqlite"):
    engine_kwargs["connect_args"] = {"check_same_thread": False}
```

- **设了环境变量** → 连 PostgreSQL
- **没设** → 自动降级 SQLite，零配置即可运行
- **团队协作**：`docker-compose.yml` 一键启动 PG

### 4.2 爬虫数据管道（去重 + 实时聚合）

```
爬虫原始数据 → find_existing_product() 去重 → upsert listing → 记录历史价格
                                                              ↓
                                          _compute_product_score()
                                          （六维加权评分：外观/物流/售后/品牌/性价比/品质）
                                                              ↓
                                          更新 Product 聚合字段
                                          (lowest/highest/historical/spread/score)
```

### 4.3 会话注入模式

```python
@router.get("/products")
def get_products(db: Session = Depends(get_db)):
    return db.query(Product).filter(...).all()
```
每个请求自动获取独立会话，请求结束自动关闭，无连接泄漏。

### 4.4 测试隔离

```python
# conftest.py
TEST_DATABASE_URL = os.getenv("TEST_DATABASE_URL", "sqlite:///./test_shopcompare.db")
engine = create_engine(TEST_DATABASE_URL, ...)
app.dependency_overrides[get_db] = override_get_db  # 替换生产会话
```
测试用独立数据库，autouse fixture 自动建表→种子→拆表，互不干扰。

## 六、关键决策与取舍

| 决策 | 选择 | 原因 |
|------|------|------|
| Product.id 用字符串非自增 | VARCHAR(20) | 多源爬虫生成 ID，避免自增冲突 |
| 聚合字段冗余 | Product 存 lowest/highest/historical | 读多写少，空间换时间 |
| 不设外键索引 | — | MVP 阶段，数据量小 |
| PriceHistory.date 用字符串 | VARCHAR(20) | 简化跨数据库兼容 |
| 不用 Alembic 迁移 | create_all() | 快速迭代，PG/SQLite 双模难一致 |
| Demo 数据归档 | archive/ | 爬虫稳定后全用爬虫数据 |

## 七、验收常见问答

**Q: 为什么不用 MySQL？**
A: PG 对 JSON、向量存储支持更好（项目中已有 pgvector 镜像）。且 SQLite 语法更接近 PG，降级时兼容性更好。

**Q: 爬虫数据质量怎么保证？**
A: save_raw_product() 有完整的去重逻辑（品牌+名称前缀匹配），每个 listing 按 product_id+platform 查重 upsert，避免重复数据。

**Q: 数据库并发怎么处理？**
A: FastAPI 每个请求独立的 Session，autocommit=False 配合显式 commit。SQLite 用 check_same_thread=False，PG 用连接池。

**Q: 数据量大了怎么办？**
A: Product.category 已有索引。下一步给 platform_listings.product_id、price_history.product_id 加索引。读写分离可将聚合计算放到写入侧（已实现）。

**Q: 怎么切换到生产 PG？**
A: 只需一行环境变量：
```bash
export DATABASE_URL=postgresql://user:pass@host:5432/shopcompare
docker compose up -d  # 如果本地开发
```
